# IAM UI Patterns

## Application View

Must show:
- Application name
- Criticality
- IAM controls
- Gaps
- Risk level

---

## Control Status

- OK → green
- ATTN → yellow
- GAP → red

---

## Risk View

Must include:
- Risk level
- Risk drivers
- Recommended actions

---

## Actions

Every gap must lead to:
- a recommendation
- a workflow

---

## Never do this:

- Show raw IAM data without meaning
- Show controls without status
- Show data without action
