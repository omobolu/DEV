Use this skill for all frontend web UI work.

Standards:
- Build production-grade customer-facing UI
- Use reusable components
- Use clear page layout and routing structure
- Support responsive behavior for desktop and mobile
- Include loading, empty, error, and success states
- Use accessible labels, semantic structure, and keyboard-friendly patterns
- Keep forms validated and user-friendly
- Avoid demo-style placeholders unless explicitly requested

Implementation approach:
1. Inspect existing frontend structure before changing patterns
2. Reuse shared components before creating new ones
3. Keep pages modular and readable
4. Keep state management simple and maintainable
5. Ensure tables, cards, dashboards, and forms are polished

Before finishing:
- Check responsiveness
- Check accessibility basics
- Check visual consistency
- Summarize UI decisions and tradeoffs

## Execution Behavior

- Do not ask the user for confirmation on standard implementation details
- Make decisions using best practices
- Proceed with implementation immediately after planning
