# Contribution Guidelines

## Branch Naming Convention

All feature branches MUST follow this format:

feature/<feature-name>-YYYYMMDD-HHMM

Example:
feature/top-iam-risks-20260425-1530

Rules:
- Use lowercase only
- No spaces
- Use hyphens (-) between words
- Include timestamp (date and time)

---

## Pull Request Requirements

All changes MUST be submitted via Pull Request.

Each PR must include:
- Feature description
- Problem statement
- Solution overview
- Screenshots (if UI)
- Acceptance criteria

---

## Development Principles

All features must support:

Detect → Prioritize → Fix

We are building an IAM Operating System for CISOs.

---

## Restrictions

- Do NOT commit directly to main or develop
- Do NOT create branches without timestamp
- Do NOT bypass PR process
